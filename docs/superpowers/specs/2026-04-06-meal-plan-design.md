# 식단표 (operation-meal.html) 화면 설계

**날짜**: 2026-04-06  
**파일**: `src/pages/oper/operation-meal.html`  
**상태**: 설계 확정 → 구현 대기

---

## 1. 개요

기관 단위 급식 식단표를 등록·조회하는 운영관리 화면.  
기관관리자가 날짜별·식사구분별(아침간식/점심/오후간식/석식)로 메뉴를 입력하고, 학부모 앱에 공유되는 식단을 관리한다. 식사구분별 반별 입력 토글을 지원한다.

---

## 2. 레이아웃 구조

```
┌─ SIDEBAR (240px) ─┬─ MAIN ────────────────────────────────┐
│ operation-        │ TOPBAR                                 │
│ sidebar.js        │ ├─ 브레드크럼: 운영관리 > 식단표       │
│ 자동 렌더          │ └─ 날짜 표시 / 새로고침                │
│                   │                                        │
│                   │ CONTENT                                │
│                   │ ├─ 월 네비게이션 바                    │
│                   │ ├─ 캘린더 영역 (월간 or 주간)          │
│                   │ └─ 상세 패널 (날짜 선택 시)            │
└───────────────────┴────────────────────────────────────────┘
```

KPI 카드 없음 — 캘린더가 콘텐츠 영역 최상단에서 바로 시작.

## 3. 캘린더 뷰

### 3-1. 월 네비게이션 바

```
[◀]  2026년 4월  [▶]                    [📅 월간으로]  ← 주간뷰일 때만 표시
```

- ◀ / ▶: 이전/다음 월(또는 주) 이동
- "📅 월간으로" 버튼: 주간뷰 → 월간뷰 복귀 시에만 노출

### 3-2. 월간 캘린더 (기본 상태)

- 7열 그리드 (월~일), 평일 셀 활성 / 토·일 비활성(흐리게)
- **각 날짜 셀** 내용:
  - 날짜 숫자
  - 등록된 식사구분 컬러 태그 (아침간식 / 점심 / 오후간식 / 석식)
  - **1개 반이라도 등록된 식사구분은 태그 표시** (반별 일부 등록도 동일)
  - 평일 중 어떤 식사구분도 없는 날: 빨간 점선 테두리 + "미등록" 레이블

| 식사구분 | 배경 | 텍스트 |
|---------|------|--------|
| 아침간식 | #D1FAE5 | #065F46 |
| 점심 | #DBEAFE | #1E40AF |
| 오후간식 | #FEF3C7 | #92400E |
| 석식 | #EDE9FE | #5B21B6 |

### 3-3. 날짜 클릭 → 주간 뷰 전환

- 클릭한 날짜가 속한 주(월~금)로 캘린더가 접힘
- 선택된 날짜 셀: #0C2D48 배경 강조, 흰 텍스트
- 각 주간 날짜 셀에 도트(●) 4색으로 식사구분 등록 여부 표시
- 캘린더 아래 상세 패널 등장

### 3-4. 주간 캘린더

- 월~금 5일 셀만 표시 (토·일 제외)
- 주 단위 ◀ ▶ 이동 가능
- 다른 날짜 셀 클릭 → 해당 날짜 상세 패널로 전환

## 4. 상세 패널

날짜 선택 시 캘린더 하단에 슬라이드다운으로 등장.

### 패널 헤더
```
📅 2026년 4월 1일 (화)  |  식단 관리
```

### 패널 바디 — 식사구분별 행 (고정 순서: 아침간식 → 점심 → 오후간식 → 석식)

각 식사구분 행은 두 가지 상태를 갖는다.

#### 등록된 식사구분
```
[아침간식]  [반별 토글]                          [✏️ 수정] [🗑 삭제]
메뉴: 바나나, 우유
알레르기: [우유]
```
- 수정·삭제 버튼: **항목 우측 상단 고정** (flex-shrink:0)
- 알레르기: 플레인 텍스트 태그 (예: `우유` `밀`)
- 이미지 첨부된 경우 썸네일 표시 (클릭 시 원본 오버레이)
- 수정·삭제: 기관관리자만 표시, 교사는 숨김

#### 미등록 식사구분
```
[오후간식]

┌ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┐  ← 점선 클릭 영역
  + 식단 입력하기                      (hover 시 #0EA5E9 테두리 + #F0F9FF 배경)
└ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ┘
```
- "+ 등록" 버튼 없음 — 점선 영역 전체가 클릭 타깃
- 클릭 시 해당 식사구분의 입력 모달 열림
- 교사 뷰에서는 점선 영역 숨김 (조회 불필요)

---

## 5. 반별 입력 토글

각 식사구분 행 우상단에 **"반별" 토글 스위치** 배치.

### 토글 OFF (기본) — 기관 공통 식단
- 하나의 메뉴 텍스트가 기관 전체에 적용

### 토글 ON — 반별 식단
- 해당 식사구분 섹션이 확장되며 기관 내 모든 반이 즉시 표시 (버튼 클릭 없이 항상 펼침)
- 상단 2px #0EA5E9 액센트 보더 + #F0F9FF 배경으로 활성 강조
- **등록된 반**: 메뉴 + 알레르기 태그 + [수정] [삭제] (우측 상단 고정)
- **미등록 반**: 점선 클릭 영역 "+ 식단 입력하기"

### 전환 규칙
| 전환 | 처리 |
|------|------|
| OFF → ON | 기존 공통 식단 삭제, 각 반 비워두고 새로 입력 |
| ON → OFF | 반별 데이터 전체 삭제 후 공통 식단 새로 입력 (경고 확인 모달 표시) |

## 6. 입력 모달 (등록 / 수정)

### 모달 헤더
- "점심 등록 — 2026년 4월 1일 (화)"
- "점심 수정 — 2026년 4월 1일 (화)"
- 반별 모드: "점심 등록 — 햇님반 — 2026년 4월 1일 (화)"

### 모달 바디

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| 메뉴 | textarea | ✅ | 자유 텍스트. 쉼표/줄바꿈 자유 |
| 알레르기 정보 | text input | ❌ | 유발 식품 나열 (예: "밀, 우유") |
| 이미지 첨부 | file input | ❌ | 식사구분당 1장. JPG/PNG/GIF, 최대 5MB |

### 모달 푸터
```
[취소]                              [저장]
```

## 11. 모킹 데이터 구조

```javascript
// 공통 식단
{ id:1, date:'2026-04-01', mealType:'점심', classId:null,
  menu:'잡곡밥, 미역국, 돈까스', allergy:'밀, 돼지고기', imageUrl:null }

// 반별 식단 (classId 있음)
{ id:2, date:'2026-04-01', mealType:'점심', classId:'CLS001',
  menu:'현미밥, 콩나물국', allergy:'', imageUrl:null }
```

`classId: null` = 공통, `classId: 'CLS001'` = 반별.  
같은 날짜·식사구분에 classId:null 과 classId 있는 레코드가 공존하지 않도록 관리.
