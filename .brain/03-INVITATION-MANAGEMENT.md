## 1. 초대장 관리 개요

초대장 관리는 **2개의 대탭**과 **학부모/교직원 서브탭**으로 구성:

```
📋 초대장 관리
├── [탭1] 초대 발송 관리
│   ├── [서브탭] 학부모 초대장
│   │   ├── 기관 선택 → 반별 초대장 관리
│   │   └── 초대 링크 생성/중지/재개
│   └── [서브탭] 교직원 초대장
│       └── 기관 선택 → 반-역할별 초대 관리
└── [탭2] 초대 이력 조회
    └── 전체 초대 이력 테이블
```

## 2. 초대 발송 관리 — 학부모 초대장

### 2.1 기관 선택 화면
- 기관 카드 그리드 형태로 표시
- 기관별 초대 현황 (전체 반 수, 활성 반 수, 총 회원 수, 대기 회원 수)
- 검색: 기관명/원장명

#### 기관 카드 데이터
```javascript
{
  id: Number,
  name: String,
  type: String,         // '어린이집'|'유치원'
  region: String,
  director: String,
  totalClasses: Number,
  activeClasses: Number,
  totalMembers: Number,
  pendingMembers: Number
}
```

### 2.2 반별 초대장 관리 (기관 진입 후)

#### 반(Class) 데이터
```javascript
{
  id: Number,
  orgId: Number,
  name: String,          // 반 이름 (햇님반, 새싹반 등)
  teacher: String,       // 담당 교사
  memberCount: Number,   // 현재 가입 학부모 수
  capacity: Number,      // 정원
  status: String,        // 'active'|'stopped'|'none'
  link: String,          // 초대 링크 URL
  toggleOn: Boolean,     // 초대 활성화 토글 상태
  members: Array<{       // 가입한 학부모 목록
    name: String,
    child: String,
    date: String,
    av: String,
    vc: String
  }>
}
```

#### 초대장 상태
| 상태 | 설명 | UI 표시 |
|------|------|---------|
| `active` | 초대 링크 활성 | 초록 뱃지, 토글 ON |
| `stopped` | 초대 일시 중지 | 노랑 뱃지, 토글 OFF |
| `none` | 초대 미설정 | 회색 뱃지, 미생성 상태 |

#### 기능
- **초대 링크 생성**: 미설정 반에 대해 새 링크 생성
- **초대 링크 복사**: 클립보드에 링크 복사
- **초대 중지/재개**: 토글로 ON/OFF 전환
- **초대 삭제**: 확인 모달 후 삭제
- **가입 학부모 목록 보기**: 반별 가입 완료 학부모 확인

## 3. 초대 발송 관리 — 교직원 초대장

### 3.1 교직원 초대 데이터
```javascript
{
  id: String,
  orgId: Number,
  cls: String,           // 반 이름
  role: String,          // '담임교사'|'보조교사'
  invitedBy: String,     // 초대 발송자
  sentAt: String,        // 발송일 (YYYY-MM-DD)
  status: String,        // 'pending'|'used'|'expired'
  dDay: Number,          // D-day (양수: 남은 일, 음수: 만료 지남)
  name: String,          // 가입자 이름 (가입 완료 시)
  link: String           // 초대 링크
}
```

### 3.2 교직원 초대 상태
| 상태 | 설명 |
|------|------|
| `pending` | 발송 완료, 가입 대기 |
| `used` | 초대 수락 → 가입 완료 |
| `expired` | 기한 만료 |

## 4. 초대 이력 조회 (탭2)

### 4.1 이력 데이터 모델
```javascript
{
  id: Number,
  status: String,        // 'active'|'used'|'expired'|'stopped'
  type: String,          // 'parent'|'teacher'
  org: String,           // 기관명
  cls: String,           // 반 이름
  name: String,          // 학부모/교사 이름
  channel: String,       // 발송 채널: '카카오'|'문자'|'링크복사'
  sentAt: String,        // 발송 일시
  joinedAt: String,      // 가입 일시 (미가입 시 빈 문자열)
  link: String,          // 초대 링크
  _sel: Boolean          // UI 선택 상태
}
```

### 4.2 KPI 카드
| 카드 | 색상 | 설명 |
|------|------|------|
| 전체 초대 | 회색 | 총 초대 발송 건수 |
| 활성(가입완료) | 초록 | 초대 수락 후 가입 완료 |
| 미가입(대기/사용완료) | 파랑 | 사용 후 아직 미가입 |
| 만료 | 빨강 | 기한 만료된 초대 |

### 4.3 이력 필터
- **상태**: 전체, 활성, 사용완료, 만료, 중지
- **유형**: 전체, 학부모, 교직원
- **기관 필터**: 기관명 검색
- **텍스트 검색**: 이름, 기관명

### 4.4 이력 테이블 컬럼
`체크박스 | 상태 | 유형 | 기관명 | 반명 | 이름 | 발송채널 | 발송일시 | 가입일시 | 초대링크 | 관리`

### 4.5 이력 관리 액션
| 액션 | 설명 |
|------|------|
| 재발송 | 초대 재발송 |
| 만료처리 | 활성 초대를 수동 만료 |
| 삭제 | 이력 삭제 |

## 5. 운영관리 초대 관리 (`operation-invitation.html`) — v2 스펙 (2026-03-24 정책 반영)

> ⚠️ **구조 변경**: 학부모 초대 단위가 반(班) → 원아(개인) 기준으로 변경됨

### 5.1 개요

기관 컨텍스트 안에서 동작. 기관 선택 단계 없음.
탭 구조: `[📩 학부모 초대] [👨‍🏫 교직원 초대] [📋 초대 이력]`

### 5.2 역할별 접근 범위

| 역할 | 학부모 초대 | 교직원 초대 | 초대 이력 |
|------|:---:|:---:|:---:|
| super (운영관리 컨텍스트) | ✅ 전체 반 | ✅ | ✅ |
| org_admin | ✅ 전체 반 | ✅ | ✅ |
| teacher | ✅ 담당 반 원아만 | ❌ (탭 비활성 + toast) | ✅ 담당 반만 |

---

### 5.3 탭1 — 학부모 초대 (원아 단위)

#### 5.3.1 리스트 구조

1행 = 초대장 1건. 반 필터 + 상태 필터 + 이름/번호 검색 제공.

| 컬럼 | 설명 |
|------|------|
| 체크박스 | 일괄 액션용 |
| 원아명 | 이름 + 반명 서브텍스트 |
| 전화번호 | 발송 번호 (미지정 시 "—") |
| 상태 뱃지 | 아래 상태표 참조 |
| 만료일 | 발송일 +7일 |
| 발송일 | 초대 생성일 |
| 액션 버튼 | 상태별 분기 |

#### 5.3.2 상태 정의

| 상태값 | 표시명 | 색상 | 액션 버튼 | 일괄 선택 포함 |
|--------|--------|------|----------|--------------|
| `inviting` | 초대중 | 파랑 | [재발송] | ✅ 만료 가능 |
| `pending_approval` | 승인요청 | 주황 | [승인] [거절] | ✅ 승인 가능 / 거절 ❌ |
| `approved` | 승인완료 | 초록 | — | — |
| `rejected` | 거절됨 | 회색 | [재초대] | — |
| `expired` | 만료됨 | 빨강 | [재초대] | — |

#### 5.3.3 일괄 액션 (체크박스 선택 후)

| 버튼 | 적용 대상 | 동작 |
|------|----------|------|
| 일괄 만료 | `inviting` 상태만 | status → `expired` |
| 일괄 승인 | `pending_approval` 상태만 | status → `approved` |

> 거절은 개별 건별 처리만 허용 (일괄 없음)

#### 5.3.4 재발송 정책

- 재발송 = 기존 링크 그대로 재발송 (링크 재생성 없음)
- 링크 재생성 조건 (아래 3가지만):
  1. 만료일이 지나 `expired` 상태에서 [재초대] 클릭
  2. 동일 전화번호로 새 초대 생성 시 (기존 링크 자동 만료 후 신규 생성)
  3. 수동 만료 처리 후 [재초대] 클릭

---

### 5.4 초대 생성 모달 (버튼 1개 → 모달 1개)

#### 5.4.1 모달 구조

```
┌─────────────────────────────────┐
│  📩 학부모 초대장 생성           │
├─────────────────────────────────┤
│  원아 선택      [드롭다운]       │  ← 전화번호 복수 입력 시 비활성
│                                 │
│  전화번호       [010-____-____]  │
│                 [+ 번호 추가]    │  ← 클릭 시 번호 입력란 추가
│  (추가된 경우)  [010-____-____] ✕│
│                 [010-____-____] ✕│
│                                 │
│  만료일         생성일 +7일 자동 │  (표시만, 수정 불가)
├─────────────────────────────────┤
│  [취소]  [링크 복사] [SMS 발송]  │  ← 전화번호 있을 때만 SMS 활성
└─────────────────────────────────┘
```

#### 5.4.2 모드 분기 로직

| 입력 상태 | 원아 선택 | 전화번호 | 동작 |
|----------|----------|----------|------|
| 원아 + 전화번호 1개 | 활성 | 1개 | SMS 발송 + 링크 복사 모두 가능 |
| 원아만 (번호 없음) | 활성 | — | 링크 복사만 가능 (SMS 버튼 비활성) |
| 전화번호 복수 | **비활성** | N개 | SMS 일괄 발송만 가능 |

#### 5.4.3 전화번호 복수 입력 시 처리

- 번호 N개 입력 → 저장 시 N개의 레코드 개별 생성
- 각 번호 기준으로 만료되지 않은 기존 링크 존재 시 → 자동 만료 후 신규 생성
- 원아 미지정 레코드는 `childName: null`, `childId: null`

### 5.5 탭2 — 교직원 초대 (v2 변경사항)

#### 5.5.1 생성 모달 변경

| 필드 | 변경 전 | 변경 후 |
|------|--------|--------|
| 대상 반 | 단일 select | **Chip 복수 선택** |
| 역할 | 담임/보조 버튼그룹 | 유지 |
| 만료일 | 없음 | **추가 (기본 7일, 수정 가능)** |

#### 5.5.2 반 복수 선택 시

- 선택된 반 수 × 역할 수 만큼 레코드 생성
- 예: 햇님반 + 달님반 × 담임 → 2개 레코드

## 6. 초대 관련 비즈니스 규칙 (v2 업데이트)

| 규칙 | 내용 |
|------|------|
| 학부모 초대 단위 | **원아 단위** (1원아 = 1초대장) |
| 기본 만료일 | 생성일 +7일 자동 설정 |
| 좌석 수 | 학부모: 고정 1 / 교직원: 설정 가능 (기본 1) |
| 재발송 | 기존 링크 재사용 (링크 재생성 없음) |
| 링크 재생성 조건 | 만료 후 재초대 / 동일 번호 재초대 / 수동 만료 후 재초대 |
| 동일 번호 중복 초대 | 기존 미만료 링크 자동 만료 → 신규 생성 |
| 번호 일괄 초대 | 원아 미지정, 번호 N개 → N개 레코드 개별 생성 |
| 교직원 반 복수 선택 | 반 수 × 역할 수만큼 레코드 생성 |
| 거절 일괄 처리 | ❌ 불가 (개별 처리만) |
